## 4. Task ID Convention

| Category | Prefix | Projects |
|----------|--------|----------|
| A — Ambition | A1-A7 | Health, Knowledge, Spiritual, Relationships, Financial, Growth, Legacy |
| B — Business | B1-B10 | Callistofx, Social, Dashboard, Infrastructure, Revenue, Marketing, Legal, Hiring, Partnerships, Product |
| C — Callings | C1+ | Side projects |

Task IDs follow the pattern: `{ProjectID}-{SequenceNumber}` (e.g., `B3-05`)

## 5. How Helios Sync Works

1. **Reads** `ACTIVE.md` (markdown tables with task rows)
2. **Reads** existing `data.json` (preserves agents, categories, briefing, etc.)
3. **Parses** tasks from ACTIVE.md table rows, mapping columns to fields
4. **Merges** new task data with existing data (never full-overwrite)
5. **Validates** all task IDs match `^[ABC]\d+-\d+$`
6. **Derives** project list and stats from tasks
7. **Checks** agent heartbeats from state files
8. **Writes** merged data.json
9. **Commits** and pushes to GitHub

**Critical rule:** Helios MERGES, never overwrites. Existing fields like `agents`, `categories`, `dailyBriefing`, `inputsNeeded` are preserved unless explicitly updated.

## 6. How to Add Tasks

### Via ACTIVE.md (primary method)

Add a row to the appropriate section table in `ACTIVE.md`:

```markdown
| B3-06 | New task title | B3 | HELIOS | 2026-04-15 |
```

Helios will pick it up on the next 15-minute sync.

### Via Telegram (Quanta agent)

Send a message to the Telegram channel. Quanta monitors and creates tasks with status `pending` and `inputsNeeded` flag.

### Via Dashboard (browser)

The "Mark Done" button in the task modal commits directly to data.json via GitHub API using a PAT stored in `localStorage.setItem('gh_pat', 'ghp_...')`.

## 7. How to Add a New Agent

1. Add to `CORE_AGENTS` dict in `helios-v3.py`
2. Create a state file at `/home/chad-yi/.openclaw/workspace/agents/{name}/state.json`:
   ```json
   {"status": "active", "lastActive": "2026-03-26T18:00:00+08:00", "currentTask": null}
   ```
3. Helios will auto-detect and include in `data.json.agents`

## 8. How to Modify the Dashboard

### Adding a new section

1. Add HTML container in `index.html` with a unique `id`
2. Add styles in `css/dashboard.css`
3. Add render function in `js/render.js` (or new file)
4. Call the render function from `Renderer.renderAll()` in `render.js`

### Modifying existing sections

- **Stats bar:** Edit `renderStatsBar()` in `js/render.js`
- **Daily briefing:** Edit `renderBriefing()` in `js/render.js`
- **Ops cards:** Edit `renderOpsCards()` in `js/render.js`
- **Focus tasks:** Edit `renderFocusTasks()` in `js/render.js`
- **Calendar:** Edit `js/calendar.js`
- **Categories/projects:** Edit `renderCategories()` in `js/render.js`
- **Modal:** Edit `js/modal.js`
- **Agents sidebar:** Edit `js/agents.js`

### CSS custom properties

All colors, spacing, typography, and radius values are defined as CSS custom properties in `:root` at the top of `css/dashboard.css`. Change the theme by editing these variables.

## 10. Polling & Data Flow

- `js/app.js` → `App.init()` runs on `DOMContentLoaded`
- Fetches `data.json` from GitHub raw URL with cache-busting `?t=` param
- Normalizes via `DashData.normalize()` (fills missing fields, filters garbage IDs)
- Renders all sections via `Renderer.renderAll()`
- Polls every 30 seconds; only re-renders if data hash changes
- Update indicator shows LOADING → SYNCED → fades out

## 11. Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| Vanilla JS, no framework | GitHub Pages only, no build step, any AI agent can edit |
| CSS custom properties | Single place to change the entire theme |
| Separate JS files | Each file has one responsibility, easy to modify |
| Merge-based sync | Previous full-overwrite bug destroyed agents/categories every 15 min |
| Task ID validation | `^[ABC]\d+-\d+$` prevents garbage rows from ACTIVE.md table separators |
| Smart polling with hash | Avoids DOM thrashing when data hasn't changed |
| Modal from anywhere | Every task click opens the same unified modal regardless of section |
